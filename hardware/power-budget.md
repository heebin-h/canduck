# 전원 계산서 (Power Budget)

> 2026-07-12: 장비 RPi5 → **RPi4(보유분)** 로 변경 확정. RPi4 peak는 ~3A로 아래 표의 RPi5 5A보다 낮음 — 표는 보수치로 그대로 두고 설계(6A급 5V 벅) 변경 없음, 마진만 커짐.

## 전체 토폴로지

```
[2S Li-Po 7.4V 5000mAh]
        │
        ├─ XT60 (배터리 인입)
        │
        ├─ [F1: 10A 폴리스위치]
        ├─ [Q1: P-MOSFET 역접속 보호]
        ├─ [Q2: Soft-start, slew-rate 제어]
        │
        ├──► [U10: 5V 6A Buck] ─── 5V_MAIN ─── RPi5, ESP32-S3, LCD BL, 센서 모듈
        │      └─ [INA219 #1]
        │
        ├──► [U11: 6V 6A Buck] ─── 6V_SERVO ── PCA9685 V+, 서보 ×10, N20 모터
        │      └─ [INA219 #2]
        │      └─ 대용량 캐패시터 뱅크 (470μF ×4 = 1880μF)
        │
        ├──► [U12: 3.3V LDO] ──── 3V3_LOGIC ── 로직 풀업, 일부 센서 보조
        │
        └─ 전압 분배기 → ADC (배터리 잔량 모니터링, ESP32 ADC)

GND: star point at 2S 리포 - 단자에서 분기. 5V/6V 레일 GND는 PCB plane에서 single bridge로 합류.
```

## 부하 리스트

### 5V_MAIN 레일

| 부하 | Idle (mA) | Active (mA) | Peak (mA) | 비고 |
|---|---|---|---|---|
| RPi 5 8GB (액티브 쿨러 포함) | 600 | 1500 | 5000 | 5V 5A PSU 권장 사양 = 25W peak |
| ESP32-S3 (Wi-Fi 비활성) | 30 | 80 | 240 | Wi-Fi 활성 시 peak↑ |
| Waveshare 1.28" LCD (백라이트 포함) | 50 | 80 | 100 | |
| PCA9685 로직 | 5 | 10 | 15 | V+는 6V 레일 |
| MPU6050 / MPR121 / INA219 ×2 / TCA9548A | 10 | 15 | 20 | I2C 디바이스 합 |
| INMP441 마이크 | 1.5 | 1.5 | 2 | I2S, 매우 저소비 |
| PAM8403 + 스피커 | 20 | 200 | 600 | 풀볼륨 시 |
| **5V 소계** | **716 mA** | **1886 mA** | **5977 mA** | |

→ **5V Buck은 6A 이상 권장**. RPi5 peak 5A 단독으로 잡아먹는 시점 + 다른 부하 동시 동작 가능성. 안전 마진 25% 더하면 7~8A급.

### 6V_SERVO 레일

| 부하 | Idle (mA) | Holding (mA) | Stall (mA) | 비고 |
|---|---|---|---|---|
| SG90 ×2 (머리) | 10 | 200 | 1400 | 2 × 700mA stall |
| MG90S ×6 (팔 4 + 다리 스윙 2) | 30 | 900 | 4200 | 6 × 700mA stall (실측은 다를 수 있음) |
| N20 6V 100RPM ×2 (휠) | 10 | 200 | 1600 | 부하 따라 변동, free 60mA / stall 800mA |
| **6V 소계 (실사용)** | **50 mA** | **1300 mA** | **7200 mA** | 동시 stall은 비현실적, 평균 활동 시 600~1500mA |

→ **6V Buck은 5A 정상 + 단시간 peak 7A 허용 설계**. 모든 서보 동시 stall은 정상 운용에서 발생하지 않지만, 시동 토크 인러시 + 동시 자세 변경 시 spike 발생. 대용량 캐패시터 뱅크 필수.

### 3V3_LOGIC 레일

I2C 풀업, MPU6050 보조, LCD 로직 일부에 쓰임. **소비 200mA 이하**, AMS1117-3.3 (1A) 또는 MIC5219 (500mA) 충분.

## 배터리 용량 / 런타임

**평균 소비 (idle + 가끔 모션)**: 5V 1.2A + 6V 0.4A ≒ 7.4V 환산 약 1.2A → **약 9W**
**활동 소비 (대화 + 머리 모션 연속)**: 5V 1.8A + 6V 1.5A ≒ 7.4V 환산 약 2.3A → **약 17W**

배터리: 2S 5000mAh = 37Wh (이론치)
실효 가용 (80% DoD 권장): 30Wh

**예상 런타임**:
- 휴면 상태 중심: 3시간+
- 적극적 활동 (대화 + 표정 + 모션 연속): 1.5~2시간
- 보행 + 풀 모션: 1시간

> 데스크탑 위 상시 운영이 목표면 충전기 항시 연결 + 배터리는 백업/이동 용도로만 운용 권장.

## BEC / Buck 선정 근거

### U10: 5V Buck (RPi5 메인)

**요구사항**: Vin 6~9V, Vout 5.1V (RPi5 PMIC 입력 마진), 6A+, 효율 90%+, low EMI

**옵션**:
- A. **Pololu D24V50F5** (15K) — 통합 모듈, 5V 5A, 효율 90%+, 검증된 보드. **추천**.
- B. **TPS54302** (1.5K) — IC 디스크리트, 직접 설계해야 함, 1.5MHz 스위칭, 5A. PCB 면적 절약 + 단가↓.
- C. **LM5161** (5K) — 100V Vin 허용, oversize지만 안정적.

→ 첫 PCB 리비전이라 **A안 (모듈 채택)** 권장. PCB에 헤더 풋프린트 박아두면 추후 디스크리트로 교체 가능.

### U11: 6V Buck (서보 레일)

**요구사항**: Vin 6~9V, Vout 6.0V, 5A 정상 / 7A peak, 큰 inrush 허용

**옵션**:
- A. **LM2596HV 모듈** (1K) — 가장 보편적, 3A 정격. 5A 부하엔 부족.
- B. **Pololu D24V60F6** (18K) — 6V 6A, 검증된 모듈. **추천**.
- C. **TPS54560** + 외부 인덕터/캡 — 5A, IC 단가 4K. 디스크리트 도전.

→ **B안** + 입력 100μF + 출력 470μF×4 캐패시터 뱅크.

### U12: 3.3V LDO

**옵션**:
- AMS1117-3.3 (0.5K) — 1A, drop-out 1.3V. Vin 5V → Vout 3.3V, 1.7V × 200mA = 340mW 발열 OK.
- MIC5219 (1K) — low drop-out, 500mA.

→ **AMS1117** 충분.

## 보호 회로

### 역접속 보호 (Q1)

```
배터리(+) ─── D ────────── 부하(+)
              │
              S──┐
                 G─── R1(10K) ── GND
                                  │
              Vbatt ── R2(100K) ─┘
```

**IRLML6402 P-channel MOSFET**: Source = 배터리(+), Drain = 부하, Gate = GND를 분압 저항으로. 정상 결선 시 Vgs 음의 큰 값 → 도통. 역접속 시 Vgs ≈ 0 → 차단. 다이오드보다 저손실.

### 저전압 차단 (UVLO)

2S 리포 컷오프 = 셀당 3.3V → 6.6V. 그 이하에서 부하 차단.

**구현**: 비교기(LM393) + 기준전압(TL431 또는 LM4040) → MOSFET 게이트 제어. 또는 BMS 내장 배터리 팩 사용 시 BMS가 알아서 차단함 (권장).

→ **BMS 내장 팩 사용 권장**, 별도 회로 생략.

### 과전류 보호

- 메인 입력: 10A 폴리스위치 (자가복귀형)
- 6V 서보 레일: 별도 5A 폴리스위치 추가
- 5V 레일: Buck IC 내장 OCP 의존

### Soft-start

큰 캐패시터 뱅크(1880μF)가 있어 인러시 전류 spike 발생 가능. Q2 (N-MOSFET + RC time constant)로 slew rate 제어 또는 NTC 인러시 리미터 (10Ω 5A) 직렬 삽입.

→ 첫 리비전은 **NTC 인러시 리미터로 단순화**, 동작 확인 후 정식 soft-start 회로 v2에 추가.

## 충전

외부 발란스 충전기 (iMAX B6 호환) 사용. 충전 단자는 XT60 + JST-XH 발란스 커넥터 노출.

**충전 중 동작**: 운영 모드와 충전 모드 분리. 데스크탑 상시 운영 시 BMS 내장 팩으로 충전과 부하 동시 가능하지만, 발열 + 셀 수명 고려 시 **충전과 운영 분리** 권장. v2에 USB-PD 직결 + 패스스루 회로 검토.

## 모니터링 / 텔레메트리

### INA219 ×2 (5V 레일 / 6V 레일)

- I2C 주소 0x40, 0x41
- Shunt: 0.01Ω (5V), 0.01Ω (6V) — 최대 8A 측정 가능
- 호스트(RPi) 측 데몬이 1Hz로 폴링 → MQTT publish `canduck/telemetry/power`

### 배터리 전압 모니터링

2S 7.4V → R1(20K)/R2(10K) 분압 → 2.47V (full), ESP32-S3 ADC 입력 (3.3V 풀스케일 안전)
ESP32 펌웨어에서 1Hz 샘플링, UART로 RPi에 보고.

### 알람 임계치

| 상태 | 임계 | 동작 |
|---|---|---|
| Vbatt < 7.0V | warning | LCD에 배터리 표정, 모션 빈도↓ |
| Vbatt < 6.6V | critical | 서보 비활성화, 슬립 모드 진입 |
| Iservo > 6A 지속 5초 | thermal warning | 모션 일시 정지 + 회복 |
| Vrail_5V < 4.8V | brownout 위험 | 즉시 모션 중단, 로깅 |

## 디커플링 / EMI 가이드

- IC 전원 핀마다 100nF 0603 디커플링 캡 (전원 핀 ≤ 5mm)
- Buck 출력 인덕터는 전류 정격 ×1.5 이상 (예: 6V 5A → 7~8A 인덕터)
- 6V/5V 레일 사이 절대 short 없도록 PCB layout 시 plane 분리
- LCD SPI 라인은 22Ω 직렬 댐핑 저항 (오버슈트 억제)
- 서보 PWM 라인은 가능하면 PCA9685 → 서보까지 짧게, 그라운드 페어링
