# Skill — Engineering Modeling on heebin's Character Files (CadQuery / FreeCAD)

> **IP 경계 (CLAUDE.md § 3, 반드시 준수)**
> - **Claude 안 함**: 공식 캐릭터(고라파덕 등) 디지털 에셋 다운로드/분석, 공식 이미지에서 비례·형태 추출, 캐릭터 외형 디자인 결정.
> - **Claude 함**: heebin이 **직접 만든** `.step`/`.stl` 파일을 받아서 그 안에서 엔지니어링만 — 파라메트릭화, 내부 공간 확보, 서보/PCB 마운트 설계, 3D 프린팅 분할, 슬라이서 호환성 검증.
> - 캐릭터 외형(비례, 실루엣, 표정 조형 등) 결정은 전부 heebin 영역. 이 문서의 작업 범위는 heebin이 최종 형상을 확정한 **이후**의 엔지니어링 단계에만 해당.

## 0. 사전 조건

- heebin이 Blender(또는 선택한 툴)로 외형 1차 모델링 완료 → `.step` 또는 `.stl` export
- CAD 엔지니어링 도구 결정 (Fusion 360 / FreeCAD+CadQuery / 기타) — Phase 2 진입 전 heebin 확정 필요 (W0-T00c 미해결 항목)

## 1. 입력 파일 인수인계

- heebin → Claude: `.step` (형상 정확, 파라메트릭 재작업 유리) 우선, 없으면 `.stl` (mesh, 치수만 참고용)
- 저장 위치 제안: `hardware/models/raw/{part}-v{n}.step` (본체/머리/팔/다리 등 파츠별)
- 전달 시 확인 사항: 스케일 단위(mm 고정), 원점 기준(바닥/무게중심 등 합의), 파츠 분리 여부

## 2. 내부 공간(cavity) 검증

목표: PCB(Board A 50×60mm + Board B 65×56mm), 배터리(2S 5000mAh 리포, 대략 75×35×20mm급), 서보 10개, 스피커/마이크가 모두 들어가는지 물리적 검증.

- [ ] 외형 껍데기의 내부 여유 공간 치수 측정 (CadQuery `BoundingBox` 또는 FreeCAD 측정 툴)
- [ ] PCB 두 장 배치 시뮬레이션 (평면 배치 또는 스택, 케이블 통과 여유 포함)
- [ ] 배터리 안착 위치 + 고정 방식 (트레이/슬롯 vs 양면테이프 자리)
- [ ] 서보 마운트 위치별 간섭 체크 (머리 2, 팔 4, 다리 2 — 관절 회전 범위 내 외형과 충돌 없는지)
- [ ] 케이블 통과홀 위치 (Board A ↔ Board B 6핀 케이블, 서보 신호선, 스피커/마이크선)
- 결과: 간섭/공간 부족 발견 시 → heebin에게 구체적 치수와 함께 리포트 (외형 수정은 heebin 몫, Claude는 "몇 mm 부족한지"까지만 명시)

## 3. 마운트 설계 (Claude 엔지니어링 영역)

- 서보 마운트: 각 서보 스펙(SG90/MG90S) 표준 마운팅 탭 치수 기준 보스/스크류홀 추가
- PCB 마운트: M2/M3 스탠드오프 위치 — Board A/B 마운트홀 좌표와 일치시켜 스크류 보스 생성
- 케이블 클립/가이드: 케이블 타이 통과용 슬롯 (필요 시)
- 원칙: 캐릭터 외형 표면 자체는 건드리지 않고, 내부에 보스/리브/슬롯을 **추가**하는 방식으로 작업 (외형 셸은 heebin 결과물 그대로 보존)

## 4. 3D 프린팅 분할

- 파츠 분리 기준: 머리(서보 마운트 포함) / 몸통 / 팔(어깨·팔꿈치 관절 마운트) / 다리(스윙 서보·휠 마운트) — roadmap.md W10 기준
- 분할 시 고려:
  - [ ] 프린터 베드 크기 내 (일반 FDM 기준 최대 파츠 치수 확인)
  - [ ] 서포트 최소화 방향으로 분할면 결정 (오버행 각도 45° 기준)
  - [ ] 조립 시 파츠 간 결합 방식 (스냅핏 / 나사 / 접착 — heebin 선호에 맞춰)
  - [ ] 분할면에 정렬 핀/홈 추가 (재조립 정확도)

## 5. 슬라이서 호환성 검증

- Export: STL (mesh) — watertight 확인 (non-manifold edge 0, 뒤집힌 노멀 0)
- Bambu Studio 또는 PrusaSlicer로 슬라이싱 시뮬레이션:
  - [ ] 벽 두께 최소값 확인 (0.2mm 노즐 기준 실제 프린트 가능 두께, 보통 ≥1.2mm 권장)
  - [ ] 서포트 필요 영역 미리보기 (내부 공간 접근 불가한 서포트는 재설계)
  - [ ] 레이어 높이 0.2mm 기준 예상 프린트 시간/재료량 산출
- 문제 발견 시 재분할 또는 리브 추가 후 재검증

## 6. 산출물 체크리스트

- [ ] 엔지니어링 반영된 `.step` (파라메트릭, 재수정 가능)
- [ ] 프린트용 `.stl` (파츠별, watertight 검증 완료)
- [ ] 치수 검증 리포트 (내부 공간 vs 부품 치수 표, 간섭 여부)
- [ ] 슬라이싱 미리보기 스크린샷 또는 요약 (서포트 위치, 프린트 시간 추정)

## 7. 외주 발주 대비 (roadmap.md 롤백 경로)

- Blender 학습이 W10 시점까지 미완이면 외주 모델링 발주 검토 (Fiverr "stylized 3D character chibi" 등, 10~20만원, ~1주)
- 이 경우도 최종 외형 파일을 heebin이 받아서 Claude에게 넘기는 흐름은 동일 — IP 경계 원칙 변하지 않음
